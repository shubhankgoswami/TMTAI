# Chat, Code & Cowork: Three Tools, Three Jobs

Most BCG users pick the wrong Claude product for the work they're doing — and then conclude "Claude isn't very good." The model is fine. The interface mismatch is the problem.

There are three Claude products and they exist because they do fundamentally different things:

**Claude Chat** (claude.ai) — a reactive thinking partner. You ask, it answers, you steer. Use it for: brainstorming, strategy exploration, drafting a memo, sanity-checking a hypothesis. It's a conversation, not a worker.

**Claude Code** — an autonomous coding agent that operates on a repository. You point it at a codebase, give it a goal, and it executes multi-step engineering work — reading files, writing code, running tests. Use it for: anything where the deliverable is code, scripts, data pipelines, or infrastructure changes.

**Claude Cowork** — a desktop application that takes goals against local files and folders. You point it at a folder, assign a task, and it executes file-level operations autonomously. Use it for: cleaning up file structures, batch-renaming, processing dozens of documents, building Excel/PowerPoint deliverables from raw inputs.

The shortcut: **Chat is for thinking. Code is for engineering. Cowork is for executing tasks on your files.**

**The watch-out for BCG:** Per BCG's RAI guidelines, client information is allowed in approved tools unless the contract or client specifically prohibits it — but never mix multiple cases in one chat (data bleed risk), and Cowork must not be used with external or untrusted Office files. When in doubt, check BCG's RAI policy before pointing any of these tools at sensitive content.
